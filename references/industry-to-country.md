# 行业 → 国家反向映射

**用法**：用户已有明确**行业方向**（"想做半导体 / 汽车 / 制药 / 咨询"）但**没决定国家**时读这个文件。

> 本文是方向性快查表。具体公司招聘门槛、签证 sponsorship 偏好年年变化——AI 给出推荐前**应主动 web search 当年招聘情况**（LinkedIn 该行业岗位密度、最近裁员 / 扩招新闻、Glassdoor 起薪）。

## 1. CS / AI / 数据科学 / 软件工程

| 国家 | 优势点 | 主要城市 / 集群 | 头部雇主举例 |
|---|---|---|---|
| 荷兰 | 工签 + 30% ruling + 英语友好 | 阿姆/埃因霍温/海牙 | Booking、ASML（部分软件岗）、Adyen、TomTom、ING |
| 德国 | 工业 + 大厂 + Blue Card 友好 | 柏林 / 慕尼黑 | SAP、Siemens、BMW（自动驾驶/AI 岗）、Zalando、Delivery Hero |
| 爱尔兰 | Big Tech 欧洲总部集中 | 都柏林 | Google、Meta、Microsoft、Stripe、AWS 欧洲 |
| 瑞典 | 独角兽密集 | 斯德哥尔摩 / 哥德堡 | Spotify、Klarna、King、Ericsson |
| 瑞士 | 顶薪 + 研究强 | 苏黎世 / 洛桑 | Google ZRH、Microsoft、ETH AI Center、各银行 quant |
| 法国 | Mistral 等新星 + 政府支持 | 巴黎 | Mistral、Hugging Face、Capgemini、各银行 |
| 东欧 | Nearshore 外包集中 | 华沙 / 布拉格 / 布加勒斯特 | 各美企 dev center |

**建议**：留欧顺位 荷兰 ≈ 爱尔兰 > 德国 > 瑞典 > 瑞士；回国看 QS 顺位反过来。

## 给 Skill 的回应原则

当用户说"我想做 X 行业去欧洲"时：
1. 先确认子方向（如"金融"——是 trading / 投行 / 量化 / 咨询？差别极大）
2. 按本表给 **2-3 个国家选项 + 各自的城市集群 + 头部雇主**
3. **主动 web search 当年招聘情况**——某些行业（如 tech）2024-2026 大幅波动，老攻略可能已失效
4. 关联回到 career-goals.md 的"留欧 vs 回国"路径：例如"汽车行业在德国就业 + 留欧极强，但回国时德国 QS 排名差异需考虑"
5. 提示用户用 LinkedIn 查目标公司在该国的招聘活跃度，作为二次验证
